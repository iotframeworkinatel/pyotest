## Key Concepts

### Test Origins
- **Registry tests**: Static tests defined in `generator/registry.py`, rendered via Jinja2 templates. `test_origin="registry"`, `test_strategy="generated"` in history.csv.
- **LLM tests**: Novel tests generated by Claude API. `test_origin="llm"`, `test_strategy="llm_generated"` in history.csv. Carry standalone `pytest_code`.

### AutoML vs LLM
- **AutoML** (H2O/AutoGluon/PyCaret/TPOT/auto-sklearn): Scores and prioritizes existing tests based on learned patterns from execution history.
- **LLM** (Claude/OpenAI/Gemini): Generates novel test code that doesn't exist in the registry. Detects coverage gaps and creates targeted tests. Provider is user-selectable via dashboard dropdown.
- They are complementary, not competing.

### LLM Provider Architecture
- Mirrors the AutoML adapter pattern: `generator/llm_providers/base.py` (abstract), `registry.py` (discover + register), per-provider adapters.
- `LLMTestGenerator(provider="claude"|"openai"|"gemini")` — delegates to the registered provider.
- Each provider implements `generate(system_prompt, user_prompt, output_schema, max_tokens) -> list[dict]`.
- Structured output: Claude uses tool use, OpenAI uses function calling, Gemini uses JSON mode with `response_schema`.
- API keys via env vars: `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`, `GOOGLE_API_KEY`.
- Models configurable via env vars: `CLAUDE_MODEL`, `OPENAI_MODEL`, `GEMINI_MODEL`.
- `GET /api/llm/providers` returns availability status for the frontend dropdown.

### Simulation Modes
- `deterministic`: No changes (baseline)
- `realistic`: Primary thesis mode — probability-driven service outages, vulnerability patches, credential rotations

## Common Tasks

### Running experiments
```bash
docker compose up -d
python run_experiments.py
```

### Building Docker images
```bash
docker compose build scanner dashboard-api
```

### Frontend dev
```bash
cd dashboard/frontend && npm run dev
```

## Important Patterns
- `TestCase.from_dict()` filters kwargs to dataclass fields — new fields with defaults are backward compatible
- Suite runner runs INSIDE the Docker scanner container; LLM API calls happen OUTSIDE (dashboard-api)
- Always derive `test_strategy` from `test_origin`, never hardcode it
- History.csv is the source of truth for hypothesis analysis
